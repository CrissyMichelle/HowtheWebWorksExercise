How the Web Works Exercise

Part One: Terminology

  • HTTP stands for HyperText Transfer Protocol and is the standard convention for web browser-to-server communications.
  • URL stands for Universal Resource Locator and contains information necessary for rendering webpage content.
  • DNS stands for Domain Name Server and is the server your router will go to when it needs help locating an IP address.
  • A query string, found at the end of a URL, requests extra information from the website by specifiying values for certain pre-defined paramters.
  • The two HTTPS methods or 'verbs' focused on so far are GET and POST. Both are requests but GET doesn't involve any side effects or impacts to the website's data. POST does involve side effects.
  • An HTTP request is what happens when your browser seeks to render a webpage.
  • An HTTP response is what is returned in consequence of an HTTP request. Both events consist of header information.
  • An HTTP header is the first part of a request or response and consists of metadata. Examples of header data I have seen include response codes and date/time info.
  • The processes that happen when you type “http://somesite.com/some/page.html” into your browser include requests and responses. First your computer searches the local cache, asks the router, and if still nothing matching the hostname is found, reaches out to the DNS. Eventually either a 404 response if returned or the IP address of somesite.com is located and your browser can start assembling the different components necessary for rendering the fully completed HTML content.
